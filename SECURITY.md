# Security Policy

## Supported versions

Security fixes are released for the latest stable version of InboxTap. Update
to the newest published version before reporting an issue that may already be
resolved.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private
vulnerability reporting form:

<https://github.com/TracyNgot/inboxtap.dev/security/advisories/new>

If the form is unavailable, contact the maintainer through the GitHub profile
to arrange a private reporting channel before sharing sensitive details.

Include the affected InboxTap version, runtime and operating system, impact,
reproduction steps, and any suggested mitigation. Do not include real email
contents, credentials, or access tokens in the report.

You should receive an acknowledgement within three business days and an
initial assessment within seven business days. Resolution timelines depend on
severity and the complexity of a safe fix. Please allow time for a release
before disclosing the issue publicly.

## Scope

Reports about the published package, CLI, SMTP capture server, HTTP API, and
client SDK are in scope. General hardening suggestions that do not describe an
exploitable vulnerability can use the public issue tracker.
